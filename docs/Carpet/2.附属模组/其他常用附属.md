# 其他常用Carpet附属模组

除了Carpet Extra和Carpet TIS Addition之外，还有许多优秀的Carpet附属模组，为玩家提供更多功能和特性。

## Carpet AMS Addition

Carpet AMS Addition是一个功能丰富的Carpet附属模组，添加了许多实用的功能。

### 主要功能

| 功能类别 | 描述 |
|----------|------|
| 游戏机制增强 | 添加新的游戏规则，扩展原版游戏机制，增强玩家体验 |
| 命令扩展 | 添加新的命令，增强现有命令功能，提供更多调试工具 |
| 性能优化 | 优化游戏性能，减少资源占用，提高服务器稳定性 |

## Carpet Redstone additions

Carpet Redstone additions专注于红石机制的扩展和增强。

### 主要功能

| 功能类别 | 描述 |
|----------|------|
| 红石组件扩展 | 添加新的红石组件，增强现有红石组件功能，提供更多红石控制选项 |
| 电路优化 | 优化红石电路性能，减少红石延迟，提高红石信号传输效率 |

## Carpet Skippy

Carpet Skippy为游戏添加了更多移动和传送功能。

### 主要功能

| 功能类别 | 描述 |
|----------|------|
| 传送功能 | 添加传送命令，增强传送机制，提供更多传送选项 |
| 移动增强 | 增强玩家移动能力，添加新的移动方式，提供更多移动控制选项 |

## 安装注意事项

### 版本兼容性

**兼容性检查：**
- 确保附属模组与主Carpet模组版本兼容
- 检查是否需要特定版本的Fabric API
- 部分附属模组可能需要Fabric Loader特定版本

### 冲突检测

**冲突避免：**
- 某些附属模组可能与其他模组存在冲突
- 避免同时使用功能重复的附属模组
- 注意附属模组之间的兼容性

### 性能影响

**性能考虑：**
- 部分功能丰富的附属模组可能会影响服务器性能
- 在生产环境中使用前应进行充分测试
- 根据服务器配置合理选择附属模组

## 使用建议

### 选择原则

**选择标准：**
- 根据服务器类型选择合适的附属模组
- 优先选择维护活跃、文档完善的模组
- 避免安装过多功能重复的附属模组

### 配置管理

**配置操作：**
- 使用`/carpet setDefault`命令保存常用配置
- 为不同世界设置不同的规则配置
- 定期备份配置文件

### 权限控制

**权限设置：**
- 合理设置命令权限等级
- 为不同用户组分配适当的规则控制权限
- 注意假玩家相关功能的权限设置
